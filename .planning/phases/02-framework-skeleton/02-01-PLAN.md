# Plan 02-01: DoML Skill Entry Points

**Phase:** 2 — Framework Skeleton
**Goal:** Create the four SKILL.md files that register DoML commands with Claude Code, making `/doml-new-project`, `/doml-plan-phase`, `/doml-execute-phase`, and `/doml-progress` invokable as slash commands.
**Requirements:** CMD-01, CMD-02, CMD-03, CMD-04, INFR-01

## Context

Claude Code discovers slash commands by reading SKILL.md files in `.claude/skills/<command-name>/SKILL.md`. Each skill has YAML frontmatter (name, description, allowed-tools) and a body that describes what the command does and references its workflow file.

The GSD framework uses the same pattern — its skills live at `.claude/skills/gsd-*/SKILL.md`. DoML mirrors this: four `doml-*` skills become the four analysis commands.

These skills are stubs in Phase 2 — the workflow files they reference will be created in Plan 02-02, and the full interview/analysis logic is implemented in Phases 3–5. Phase 2 just ensures the commands are registered and invokable.

<objective>
Register four DoML slash commands with Claude Code by creating SKILL.md files.

Purpose: Data scientists can invoke `/doml-new-project` etc. immediately after framework installation.
Output: Four `.claude/skills/doml-*/SKILL.md` files, each pointing to its workflow file.
</objective>
