# Plan 20-02: `install.ps1` — PowerShell Install Script

## Goal

Create `install.ps1` at the DoML repo root. PowerShell equivalent of `install.sh` — same archive-based download, same file tree installed, same idempotency rules, same fail-fast behavior. Targets Windows PowerShell 5.1+ and PowerShell Core 7+. Uses `bsdtar` (built into Windows 10 1803+) for `.tar.gz` extraction.

## File Created

- `install.ps1` (repo root)

## Decisions Applied

- D-01: Single `.tar.gz` archive download via `Invoke-WebRequest`
- D-02: Same `.tar.gz` as Bash (Windows ships bsdtar)
- D-03: Install into CWD; no directory argument
- D-04: Same file set as install.sh
- D-05: `-Version` parameter; `$env:DOML_VERSION` fallback for `irm ... | iex` pipe mode
- D-06: Same idempotency rules
- D-07: Fail-fast in `catch` block; temp dir removed in `finally`
- D-08: CLAUDE.md from archive root

## Key Patterns

- `#Requires -Version 5.1` — built-in version gate
- `$ErrorActionPreference = "Stop"` — terminating errors
- `try { ... } finally { Remove-Item -Recurse -Force $TempDir }` — guaranteed cleanup
- `Invoke-WebRequest -UseBasicParsing` — avoids IE engine dependency on PS 5.1
- `& tar -xzf ...; if ($LASTEXITCODE -ne 0)` — external process exit code check
- `Get-ChildItem -Filter "doml-*" | foreach { Copy-Item -Force }` — skill enumeration
- `$env:DOML_VERSION` fallback for version pin in pipe-to-iex mode
